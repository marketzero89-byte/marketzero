# 05 — Data Infrastructure

---

## Data Sources

| Source | Type | Symbols | Frequency |
|---|---|---|---|
| Alpaca Markets | OHLCV equity bars | AAPL, SPY, GLD, + any US equity | 1-min / 1-day |
| Alpaca Crypto | OHLCV crypto bars | BTC/USD, ETH/USD | 1-min |
| Internal simulator | Synthetic prices | All | Per-step (paper mode) |

In paper mode, prices are generated by the internal `PaperBroker` using a Geometric Brownian Motion simulator with configurable drift and volatility per symbol.

---

## Market Data Pipeline

```
Alpaca REST/WebSocket
        │
        ▼
  get_latest_price(symbol)    ← called every step in trading loop
        │
        ▼
  RegimeDetector.update()     ← maintains rolling price deque (maxlen=100)
        │
        ▼
  Agent.observe(price)        ← each agent builds its own feature vector
```

---

## Price Data Schema

Each agent receives a flat observation vector built from recent prices. The observation window is controlled by the `lookback` hyperparameter (5–60 bars, evolved per agent).

```
observation = [
    price_t,
    price_t-1,
    ...
    price_t-lookback,
    rsi_value,          # RSI(rsi_period)
    bb_upper,           # Bollinger upper band
    bb_lower,           # Bollinger lower band
    atr_value,          # ATR(atr_period)
    regime_encoded,     # 0=UNKNOWN, 1=BULL, 2=BEAR, 3=RANGING, 4=HIGH_VOL
]
```

## Storage Layout

```
logs/
├── pbt_orchestrator.json     # JSON-lines: one record per generation step
├── pbt_genealogy.json        # Agent lineage: who copied from whom
├── leaderboard.html          # Static leaderboard snapshot (last gen)
└── tensorboard/              # TensorBoard event files (if enabled)

checkpoints/
└── generation_NNNN/
    ├── population.pkl        # Full agent population state
    ├── config.json           # Config snapshot at that generation
    └── metrics.json          # Fitness scores at checkpoint time
```

## Log Record Format

Each line in `pbt_orchestrator.json` follows this schema:

```json
{
  "step": 42,
  "timestamp": 1718000000.0,
  "generation": 5,
  "avg_fitness": 0.312,
  "best_fitness": 0.847,
  "portfolio_equity": 125400.0
}
```

## Data Retention Policy

| Data Type | Retention |
|---|---|
| JSON logs | Rolling 30 days (manual cleanup) |
| Checkpoints | Last 10 generations kept automatically |
| TensorBoard events | Rolling 7 days |
| Backtest CSVs | Permanent (timestamped filenames) |

## Extending Data Sources

To add a new data provider, implement the broker interface in `trading/live_trading.py`:

```python
class MyBroker:
    def get_latest_price(self, symbol: str) -> float: ...
    def get_portfolio_state(self) -> PortfolioState: ...
    def submit_order(self, symbol, qty, side, confidence) -> dict: ...
    def get_positions(self) -> dict: ...
```

Set `broker_mode` to `"custom"` and pass your broker instance to `PBTLiveExecutor`.

## Dataset Directory

The `datasets/` directory is reserved for:
- Historical OHLCV CSV files used in backtests
- Alternative data sources (sentiment, options flow)
- Synthetic datasets generated for unit tests

All files should be named: `{SYMBOL}_{YYYY-MM-DD}_{frequency}.csv`

Example: `AAPL_2024-01-01_1min.csv`
